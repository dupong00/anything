# 🗳️ anything : 실시간 메뉴 결정 시스템

> "오늘 뭐 먹지?"라는 고민을 실시간 투표와 데이터 기반 추천으로 명쾌하게 해결하는 서비스 > 단순한 투표를 넘어 근처 식당을 추천해주는 대규모 트래픽과 확장성을 고려한 Modular Monolith 아키텍처 프로젝트입니다.
> 

---

## 🚀 프로젝트 배경

단체 모임이나 직장에서 메뉴를 정할 때 발생하는 결정 장애를 해결하기 위해 기획되었습니다. 사용자는 실시간으로 메뉴 후보를 제안하고 투표 결과가 즉각적으로 반영되는 환경을 경험합니다.

이 프로젝트의 핵심은 성장 가능한 구조입니다. 비즈니스 초기에는 빠른 개발 속도를 유지하되 트래픽 증가 시 특정 도메인을 즉시 MSA로 분리할 수 있는 설계를 지향합니다.

## 🏗️ 아키텍쳐 : Modular Monolith

본 프로젝트는 일반적인 계층형 아키텍처(Layered Architecture) 대신 **모듈러 모놀리스** 구조를 채택했습니다.

### Why Modular Monolith?

- **도메인 간 낮은 결합도:** `Vote(투표)`, `Menu(메뉴)`, `User(사용자)` 모듈을 물리적으로 격리하여 각 도메인의 비즈니스 로직이 섞이는 것을 방지했습니다.
- **MSA 전환 전략:** 실시간 투표(Vote) 모듈은 트래픽이 집중되는 특성이 있습니다. 향후 서비스 확장 시, 코드의 재설계 없이 해당 모듈만 독립적인 마이크로서비스로 배포할 수 있는 구조를 갖췄습니다.
- **인프라 복잡도 최소화:** 처음부터 완전한 MSA로 시작할 때 발생하는 분산 트랜잭션 및 네트워크 비용을 줄이면서도, 논리적인 서비스 분리를 통해 유지보수성을 극대화했습니다.

---

## ✨ 핵심 기능

- **실시간 투표 :** WebSocket을 활용해 투표 현황이 실시간으로 모든 사용자에게 브로드캐스팅됩니다.
- **메뉴 추천 :** 투표 결과를 바탕으로 사용자 위치 기반 식당 정보를 추천합니다.
- **동시성 관리 :** 짧은 시간 내 집중되는 투표 요청을 Redis를 활용해 안정적으로 처리합니다.

## ⚙️ Getting Started

### 필수 설정

본 프로젝트는 보안을 위해 환경 변수를 사용합니다. 실행 전 아래 항목을 설정해 주세요.

- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`
- `JWT_SECRET`
- `JWT_EXPIRE`
- `JWT_REFRESH_EXPIRE`
